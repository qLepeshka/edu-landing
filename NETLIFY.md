# 🔥 ПОДРОБНАЯ ИНСТРУКЦИЯ ПО NETLIFY

## Почему ваш сайт не работает на Netlify?

Есть несколько частых причин. Давайте разберём каждую и исправим.

---

## 📋 ПОШАГОВАЯ ИНСТРУКЦИЯ (С ФОТО)

### ШАГ 1: Регистрация/Вход

1. Откройте браузер
2. Перейдите на **https://app.netlify.com**
3. Нажмите **"Sign up"** (если нет аккаунта) или **"Log in"**
4. Выберите способ регистрации:
   - **Email** — введите почту и создайте пароль
   - **GitHub** — если есть аккаунт GitHub (рекомендуется)
   - **GitLab** — если есть GitLab
   - **Bitbucket** — если есть Bitbucket

---

### ШАГ 2: Создание сайта

**СПОСОБ А: Drag & Drop (Самый простой)**

1. После входа вы увидите **"Sites"** в левом меню
2. Нажмите **"Add new site"** → **"Deploy manually"**
3. Откроется страница с большой областью для загрузки
4. **Перетащите ПАПКУ** `лендинг` целиком в эту область
5. Подождите 10-30 секунд
6. Сайт опубликован! Вы увидите URL вида `random-name-12345.netlify.app`

**ВАЖНО:** Перетаскивайте именно **ПАПКУ**, а не файлы по отдельности!

---

**СПОСОБ Б: Через Netlify CLI (Для продвинутых)**

1. Откройте командную строку (PowerShell или CMD)

2. Установите Netlify CLI:
```bash
npm install -g netlify-cli
```

3. Перейдите в папку с проектом:
```bash
cd C:\РАБОТА\лендинг
```

4. Войдите в аккаунт:
```bash
netlify login
```
Откроется браузер — нажмите "Authorize"

5. Создайте сайт:
```bash
netlify init
```
   - Выберите "Create & configure a new site"
   - Выберите команду (Team): ваше имя
   - Введите название сайта: `evo-landing` (или любое другое)
   - Build command: оставьте пустым (нажмите Enter)
   - Publish directory: `.` (точка)

6. Опубликуйте:
```bash
netlify deploy --prod
```

---

### ШАГ 3: Проверка работы сайта

1. После деплоя вы получите URL вида:
   ```
   https://evo-landing-abc123.netlify.app
   ```

2. Откройте эту ссылку в браузере

3. **Проверьте:**
   - ✅ Загружается ли страница?
   - ✅ Видны ли все секции?
   - ✅ Работают ли кнопки?
   - ✅ Открывается ли чат?

## ❌ ЧАСТЫЕ ПРОБЛЕМЫ И ИХ РЕШЕНИЯ

### Проблема 1: "Page Not Found" (404)

**Причина:** Netlify не нашёл `index.html`

**Решение:**
1. Убедитесь, что файл называется именно `index.html` (не `Index.html`, не `INDEX.HTML`)
2. Файл должен быть в **КОРНЕ** папки, а не в подпапке

**Проверьте структуру:**
```
лендинг/
├── index.html      ← ДОЛЖЕН БЫТЬ ЗДЕСЬ
├── styles.css
├── script.js
└── favicon.svg
```

**Если файлы в подпапке:**
- Переместите `index.html` в корень папки
- Загрузите заново

---

### Проблема 2: Сайт загружается, но стили не работают

**Причина:** Неправильные пути к CSS/JS файлам

**Решение:**
1. Откройте `index.html`
2. Найдите строки:
```html
<link rel="stylesheet" href="styles.css">
<script src="script.js"></script>
```

3. Убедитесь, что пути **ОТНОСИТЕЛЬНЫЕ** (не абсолютные):
   - ✅ `href="styles.css"` — правильно
   - ❌ `href="C:\РАБОТА\лендинг\styles.css"` — неправильно
   - ❌ `href="/styles.css"` — может не работать

---

### Проблема 3: Белый экран

**Причина:** Ошибка в JavaScript или не загружены файлы

**Решение:**
1. Откройте консоль браузера (F12 → Console)
2. Посмотрите, есть ли красные ошибки
3. Частые ошибки:
   - Файл `script.js` не загружен — проверьте имя файла
   - Синтаксическая ошибка — проверьте код

---

### Проблема 4: "Deploy failed"

**Причина:** Проблема с загрузкой файлов

**Решение:**
1. Проверьте размер файлов — не должно быть очень больших (>50MB)
2. Убедитесь, что все файлы в папке
3. Попробуйте загрузить заново:
   - Удалите сайт в Netlify (Site settings → Danger Zone → Remove site)
   - Создайте новый и загрузите папку снова

---

### Проблема 5: Кэш — изменения не видны

**Причина:** Браузер кэширует старую версию

**Решение:**
1. Откройте сайт в режиме инкогнито (Ctrl+Shift+N)
2. Или очистите кэш:
   - Chrome: Ctrl+Shift+Delete → Cache images and files → Clear
3. Или используйте "жесткую перезагрузку": Ctrl+F5

## 🔧 ДЕПЛОЙ С GITHUB (АВТОМАТИЧЕСКИЙ)

Этот способ лучше — при каждом изменении код сайт обновляется автоматически.

### Шаг 1: Создание репозитория

1. Откройте https://github.com
2. Нажмите "+" → "New repository"
3. Название: `evo-landing`
4. Public
5. "Create repository"

### Шаг 2: Инициализация Git

Откройте командную строку в папке лендинга:

```bash
cd C:\РАБОТА\лендинг
git init
git add .
git commit -m "Initial commit: EvoPlatform landing"
git branch -M main
git remote add origin https://github.com/ВАШ_НИК/evo-landing.git
git push -u origin main
```

**Если просит логин/пароль:**
- Логин: ваш ник GitHub
- Пароль: Personal Access Token (создайте в GitHub Settings → Developer settings → Personal access tokens)

### Шаг 3: Подключение к Netlify

1. Войдите в Netlify
2. "Add new site" → "Import an existing project"
3. Выберите GitHub
4. Авторизуйте доступ
5. Выберите репозиторий `evo-landing`
6. Branch: `main`
7. Build command: оставьте пустым
8. Publish directory: оставьте пустым
9. Нажмите "Deploy site"

**ГОТОВО!** Теперь при каждом `git push` сайт будет обновляться автоматически!

## 🔗 СВОЙ ДОМЕН НА NETLIFY

Если хотите адрес типа `www.mysite.ru` вместо `mysite.netlify.app`:

### Шаг 1: Купите домен
- https://reg.ru (от 199₽/год)
- https://nic.ru

### Шаг 2: Добавьте в Netlify
1. Site settings → Domain management
2. "Add custom domain"
3. Введите: `www.mysite.ru`
4. Нажмите "Verify"

### Шаг 3: Настройте DNS
В панели управления доменом (например, reg.ru):

1. Найдите "Управление DNS" или "DNS-серверы"
2. Добавьте CNAME запись:
   - Host: `www`
   - Value: `evo-landing-abc123.netlify.app`

3. Или A запись:
   - Host: `@`
   - Value: `75.2.60.5` (IP Netlify)

**Ждите:** DNS обновляются 1-24 часа
